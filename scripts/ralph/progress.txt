## Codebase Patterns
- Unified LLM client at server/app/tools/llm_client.py — all OpenAI calls must go through llm_client.chat(); auto-records token usage via asyncio.create_task() fire-and-forget
- SQLite UPSERT pattern: use `from sqlalchemy.dialects.sqlite import insert as sqlite_insert` with `.on_conflict_do_update(index_elements=[...], set_={...})`
- Admin services live in server/app/services/admin/ (separate subpackage from main services/)
- New model noqa imports must be added to main.py for create_all to discover them (pattern: `import app.models.X  # noqa: F401`)
- asyncio.create_task() works inside FastAPI endpoints for fire-and-forget background jobs; use dns.asyncresolver (not dns.resolver) for async DNS MX checks
- Union-Find (path compression) is an efficient way to group similar items; use defaultdict(list) for grouping by root
- difflib.SequenceMatcher(None, a, b).ratio() gives string similarity 0.0-1.0; group by platform first to reduce O(n²) comparisons
- Admin influencers router prefix: /api/admin/influencers; registered in main.py after admin_mailboxes_router
- Recharts PieChart with innerRadius creates donut charts; Cell order matches data array order
- EmailStatus enum is VARCHAR in SQLite — new values (queued, blocked, cancelled) can be added to Python enum without ALTER TABLE
- Admin emails API prefix: /api/admin/emails; blacklist sub-resource: /api/admin/emails/blacklist
- sender_service.py is a new services file; actual send loop lives in agents/sender.py (Sender Agent)
- Backend runs from server/ directory; database URL uses relative path ./data/influencer.db, so data/ must exist under server/
- All models import Base from app.database, not from sqlalchemy directly
- Config uses pydantic-settings BaseSettings with @lru_cache; import via get_settings()
- SQLAlchemy 2.0 mapped_column + Mapped[] style (not old Column style)
- FastAPI app uses lifespan context manager for startup/shutdown (not deprecated on_event)
- WebSocket manager is a singleton in app/websocket/manager.py
- Frontend proxy: /api → http://localhost:8000, /ws → ws://localhost:8000
- Pydantic v2: use model_validate() (not from_orm()), model_dump(exclude_unset=True) for PATCH
- Enum fields on SQLAlchemy models: read .value for str output (e.g. inf.status.value)
- New API router: add import + app.include_router() in main.py; prefix="/api" at register site
- Admin API routers nested under server/app/api/admin/; registered in main.py with prefix="/api/admin"
- require_admin dependency already exists in server/app/deps.py (checks role == "admin", raises 403)
- Frontend admin components in client/src/components/admin/; pages in client/src/pages/admin/
- Role-based post-login redirect: parse JWT from localStorage after login(), check role field
- prd.json top-level key is "userStories" (not "stories")
- token_version field in User model + JWT payload ("tv" claim) enables force-logout by incrementing DB value
- get_current_user is now async and queries DB on every request to validate token_version (one extra query per auth'd request)
- write_login_history() creates its own AsyncSession to avoid transaction rollback on login failures
- ALT TABLE migrations in main.py lifespan use try/except silently (column may already exist)
- Middleware directory: server/app/middleware/; register via app.add_middleware() in main.py AFTER CORSMiddleware
- asyncio.create_task() inside BaseHTTPMiddleware.dispatch() for fire-and-forget background DB writes
- Unit tests live in server/tests/; run from server/ directory with `python -m pytest tests/`
- pytest + pytest-asyncio are in requirements.txt; asyncio mode=STRICT requires @pytest.mark.asyncio
