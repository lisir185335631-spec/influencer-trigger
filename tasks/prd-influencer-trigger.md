> **产品名称**：Influencer Trigger System
> **版本**：v1.0 MVP
> **日期**：2026-04-13
> **状态**：Draft

---

## 1. 产品概述

### 1.1 一句话定义
为跨境电商团队打造的 AI Agent 系统，自动从海外社交媒体平台抓取网红邮箱、批量发送合作邮件、实时监控邮件状态、智能分流回复，并提供 CRM 式网红管理能力。

### 1.2 核心价值
- **效率**：5-10 人团队无需逐一查找网红联系方式，系统自动批量抓取 + 发送
- **不遗漏**：全量触达，先发后筛，让市场数据说话
- **持续触达**：无回复网红按月自动追发 + 节假日祝福，保持品牌存在感
- **快速响应**：有回复实时通知，人工第一时间介入高意向网红

### 1.3 目标用户
| 角色 | 职责 | 使用场景 |
|------|------|---------|
| BD 运营 | 日常操作系统，配置任务，处理回复 | 创建抓取任务、设置邮件模板、跟进回复网红 |
| BD 主管 | 监控整体进度，决策优先级 | 查看仪表盘、审核筛选结果、分配跟进任务 |
| 管理员 | 系统配置，邮箱管理 | 添加发送邮箱、配置平台参数、管理团队账号 |

### 1.4 目标平台
TikTok、Instagram、YouTube、Twitter（X）、Facebook

## 2. 业务流程

### 2.1 端到端流程图

```
用户输入行业关键词 + 选择平台
          │
          ▼
  ┌───────────────┐
  │ Scraper Agent │ ← Playwright 无头浏览器
  │ 抓取网红邮箱   │
  └───────┬───────┘
          │ 100-200 条/批次
          ▼
  ┌───────────────┐
  │ Sender Agent  │ ← 多邮箱轮换发送
  │ 批量发送邮件   │   SendGrid/SMTP
  └───────┬───────┘
          │
          ▼
  ┌───────────────┐     ┌──────────────┐
  │ Monitor Agent │────→│  有回复       │──→ Classifier Agent（意图分类）
  │ 24/7 监控状态  │     │              │    ──→ 系统通知 BD 运营
  └───────┬───────┘     └──────────────┘    ──→ 人工在自己邮箱跟进
          │
          ▼
  ┌──────────────┐
  │  无回复       │──→ 每月自动追发 1 次
  │              │──→ 节假日自动发送祝福邮件
  │              │──→ 超过 N 月无回复 → 归档
  └──────────────┘
```

### 2.2 核心业务规则

| 规则 | 说明 |
|------|------|
| 全量触达 | 抓取到邮箱即发送，不做前置筛选 |
| 多邮箱轮换 | 用户配置多个发送邮箱，系统自动轮换，单邮箱每日上限可配置（默认 50 封） |
| 月度追发 | 无回复网红每月自动追发 1 次，追发内容差异化 |
| 节假日祝福 | 内置国际节假日日历（圣诞、新年、感恩节、情人节等），到期自动发送祝福邮件 |
| 回复即转人工 | 检测到回复后，不做自动跟进，直接通知人工在自己邮箱处理 |
| 行业自适应 | 用户输入行业关键词，LLM 动态生成匹配的邮件模板和网红评估维度 |

### 3.2 M1: 网红邮箱抓取

#### 功能描述
用户输入行业关键词和目标平台，系统自动从社交媒体平台抓取网红公开邮箱信息。

#### 用户故事

**US-1.1 创建抓取任务**
> 作为 BD 运营，我要能创建一个抓取任务（选择平台 + 输入行业关键词 + 设置抓取数量），系统自动执行抓取并返回结果。

验收标准：
- 支持选择 1-5 个平台同时抓取
- 输入行业关键词（如 "tattoo equipment"、"beauty skincare"）
- 设置目标抓取数量（默认 100，上限 200）
- 任务提交后显示进度条（已抓取/目标数）
- 抓取完成后显示结果列表

**US-1.2 网红信息提取**
> 作为系统，抓取时要同时提取网红的基础画像信息，用于后续 CRM 管理和筛选。

验收标准：
- 提取字段：昵称、平台、邮箱、主页链接、粉丝数、简介、地区（如可获取）
- 邮箱来源：bio / about / contact 页面 / 外链
- 反混淆处理：`[at]` → `@`、`(dot)` → `.`、空格干扰去除
- MX 记录验证：过滤无效邮箱域名
- 跨平台去重：同一邮箱只保留粉丝最多的一条

**US-1.3 抓取结果预览**
> 作为 BD 运营，抓取完成后我要能预览结果列表，确认无误后一键进入发送流程。

验收标准：
- 表格展示：网红名 / 平台 / 邮箱 / 粉丝数 / 简介摘要
- 支持按粉丝数排序
- 支持手动移除不想联系的网红
- "全部发送"按钮进入邮件发送流程

**US-1.4 导入已有网红名单**
> 作为 BD 运营，我要能导入已有的网红名单（Excel/CSV），直接进入发送流程，无需抓取。

验收标准：
- 支持 .xlsx 和 .csv 格式上传
- 必填列：邮箱（email）；选填列：昵称、平台、粉丝数、主页链接、行业
- 列名自动匹配（支持中英文列名，如"邮箱"="email"）
- 导入前预览：展示前 10 行 + 列映射确认
- 邮箱格式校验 + MX 验证，无效邮箱标红提示
- 去重：与系统已有网红邮箱自动去重，重复项提示并可选跳过或覆盖
- 导入完成后进入网红列表，可直接选择"全部发送"

### 3.3 M2: 邮件发送引擎

#### 功能描述
将抓取到的网红邮箱列表批量发送合作邮件，支持多邮箱轮换和模板管理。

#### 用户故事

**US-2.1 发送邮箱管理**
> 作为管理员，我要能添加和管理多个发送邮箱（SMTP 配置），系统自动轮换使用。

验收标准：
- 添加邮箱：输入邮箱地址 + SMTP 服务器 + 端口 + 密码/授权码
- 连接测试：添加后自动发送测试邮件验证配置
- 每日上限：每个邮箱可配置每日发送上限（默认 50 封）
- 轮换策略：均匀分配，当某邮箱达到日限额自动切换下一个
- 邮箱状态：正常 / 达到限额 / 配置错误 / 已禁用
- 支持至少 10 个发送邮箱

**US-2.2 邮件模板管理**
> 作为 BD 运营，我要能管理邮件模板，支持 LLM 根据行业自动生成模板。

验收标准：
- 模板变量：`{{influencer_name}}`、`{{platform}}`、`{{followers}}`、`{{industry}}`
- LLM 生成：输入行业关键词，AI 自动生成 3 套不同风格的邮件模板（正式/轻松/直接）
- 手动编辑：用户可编辑 AI 生成的模板或从零创建
- 模板库：保存常用模板，按行业分类
- 预览：发送前可预览变量替换后的实际邮件内容
- 支持 HTML 富文本格式
- 模板语言仅英文（目标受众为海外网红）

**US-2.3 批量发送执行**
> 作为 BD 运营，确认网红列表和模板后，一键启动批量发送。

验收标准：
- 选择网红列表 + 选择邮件模板 → 确认发送
- 发送进度实时展示（已发送/总数/成功/失败）
- 多邮箱自动轮换，用户无感知
- 发送间隔可配置（默认每封间隔 30-60 秒随机）
- 失败自动重试 1 次，仍失败则标记并跳过
- 发送完成后生成发送报告

### 3.4 M3: 邮件状态监控

#### 功能描述
24/7 不间断监控所有已发送邮件的状态变化，实时更新。

#### 用户故事

**US-3.1 邮件状态追踪**
> 作为系统，需要持续监控每封邮件的生命周期状态。

验收标准：
- 状态枚举：已发送 → 已送达 → 已打开 → 已回复 / 未回复 / 退信
- 监控方式：
  - IMAP 轮询：每 5 分钟检查收件箱是否有回复邮件
  - SMTP 退信检测：解析退信通知邮件
- 状态变更实时写入数据库
- WebSocket 推送状态变更到前端
- 24/7 不间断运行，异常自动恢复

**US-3.2 邮件状态看板**
> 作为 BD 运营，我要在系统内看到所有邮件的实时状态。

验收标准：
- 状态汇总卡片：总发送数 / 已送达 / 已打开 / 已回复 / 未回复 / 退信
- 邮件列表：按状态筛选，显示网红名 / 邮箱 / 发送时间 / 当前状态 / 最后更新时间
- 实时更新：状态变更时页面自动刷新（WebSocket 推送）
- 支持按发送批次、平台、状态筛选

### 3.5 M4: 智能分流与通知

#### 功能描述
检测到网红回复后，AI 分类回复意图，系统通知人工介入。

#### 用户故事

**US-4.1 回复检测与意图分类**
> 作为系统，检测到回复邮件后，AI 自动分析回复内容的意图。

验收标准：
- 意图分类（LLM 驱动）：
  - 感兴趣（想了解更多 / 询问合作细节）
  - 报价咨询（直接问价格 / 佣金）
  - 婉拒（礼貌拒绝 / 暂时不需要）
  - 自动回复（out of office / auto-reply）
  - 无关（垃圾回复 / 不相关内容）
- 分类结果写入 CRM 记录
- 自动回复（out of office）不触发人工通知

**US-4.2 人工介入通知**
> 作为 BD 运营，网红回复后我要立即收到通知，去自己的邮箱跟进。

验收标准：
- 通知方式：
  - 系统内通知（实时弹窗 / 通知栏）
  - 可选：邮件通知 / 飞书/Slack Webhook
- 通知内容：网红名 / 平台 / 回复意图分类 / 回复内容摘要
- 通知中标明"请前往您的邮箱回复"
- 按意图优先级排序：感兴趣 > 报价咨询 > 婉拒
- 支持标记"已处理"

### 3.6 M5: 自动追发引擎

#### 功能描述
对未回复网红按策略自动追发邮件，包括月度追发和节假日祝福。

#### 用户故事

**US-5.1 月度自动追发**
> 作为系统，对未回复的网红每月自动追发 1 封邮件。

验收标准：
- 追发间隔：每 30 天 1 次
- 追发内容：每次不同（LLM 生成差异化内容，换角度、换价值主张）
- 追发邮箱：继续使用原发送邮箱（保持发件人一致性）
- 追发上限：可配置最大追发次数（默认 6 次，即半年）
- 超过上限 → 自动归档，不再追发
- 如果中途网红回复，立即停止追发，进入人工介入流程

**US-5.2 节假日祝福邮件**
> 作为系统，在国际节假日自动向全部联系过的网红发送祝福邮件。

验收标准：
- 内置节假日日历：
  - 圣诞节（12/25）、新年（1/1）、感恩节（11月第4个周四）
  - 情人节（2/14）、万圣节（10/31）、复活节
  - 用户可自定义添加节日
- 祝福模板：LLM 根据节日 + 行业生成个性化祝福
- 发送范围：所有状态非"归档"的网红
- 祝福邮件不计入追发次数
- 如果当月已有追发/祝福，不重复发送

### 3.7 M6: CRM 网红管理

#### 功能描述
提供 CRM 式的网红管理功能，支持标签、备注、合作历史记录。

#### 用户故事

**US-6.1 网红档案**
> 作为 BD 运营，我要能查看每个网红的完整档案。

验收标准：
- 基础信息：昵称、平台、邮箱、粉丝数、简介、主页链接、头像
- 邮件历史：所有往来邮件时间线（发送 / 回复 / 追发）
- 状态标签：待联系 / 已联系 / 已回复 / 合作中 / 已归档
- 自定义标签：用户可创建和分配标签（如"高意向"、"大V"、"已签约"）
- 备注：自由文本备注，支持多条，按时间排序
- 合作历史：记录合作次数、合作内容、合作金额（手动录入）

**US-6.2 网红列表与筛选**
> 作为 BD 运营，我要能按多维度筛选和搜索网红。

验收标准：
- 筛选维度：平台 / 状态 / 标签 / 粉丝数区间 / 行业 / 回复意图
- 关键词搜索：支持搜索昵称、邮箱、备注内容
- 批量操作：批量打标签、批量归档、批量导出
- 导出：导出为 CSV（网红名 / 邮箱 / 平台 / 状态 / 标签）

**US-6.3 回复网红筛选**
> 作为 BD 主管，我要对已回复的网红进行质量评估和优先级排序。

验收标准：
- 评估维度（LLM + 规则混合）：
  - 合作意向（来自回复意图分类）
  - 粉丝量级（来自抓取数据）
  - 内容匹配度（LLM 分析网红内容与行业的相关性）
- 优先级评分：高 / 中 / 低
- 排序列表：按优先级排序，高意向网红排最前
- BD 主管可手动调整优先级

### 3.8 M7: 数据仪表盘

#### 功能描述
展示整体运营数据，辅助团队决策。

#### 用户故事

验收标准：
- 核心指标卡片：
  - 总网红数 / 本周新增

系统配置和团队管理。

## 4. 数据模型

### 4.1 核心实体

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│ influencers │────→│   emails    │────→│email_events │
│ (网红档案)   │     │ (邮件记录)  │     │ (状态事件)  │
└──────┬──────┘     └─────────────┘     └─────────────┘
       │
       ├───→ tags (标签)
       ├───→ notes (备注)
       └───→ collaborations (合作历史)

┌─────────────┐     ┌─────────────┐
│ mailboxes   │     │ templates   │
│ (发送邮箱)  │     │ (邮件模板)  │
└─────────────┘     └─────────────┘

┌─────────────┐     ┌─────────────┐
│ campaigns   │     │ holidays    │
│ (发送批次)  │     │ (节假日)    │
└─────────────┘     └─────────────┘

┌─────────────┐     ┌─────────────┐
│   users     │     │ scrape_tasks│
│ (团队成员)  │     │ (抓取任务)  │
└─────────────┘     └─────────────┘
```

### 4.2 关键表结构

#### influencers（网红档案）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| nickname | VARCHAR(200) | 网红昵称 |
| platform | ENUM | tiktok/instagram/youtube/twitter/facebook |
| email | VARCHAR(200) | 邮箱（唯一索引） |
| profile_url | TEXT | 主页链接 |
| followers | INTEGER | 粉丝数 |
| bio | TEXT | 简介 |
| region | VARCHAR(100) | 地区 |
| industry | VARCHAR(100) | 所属行业 |
| status | ENUM | scraped/contacted/replied/cooperating/archived |
| reply_intent | ENUM | interested/pricing/declined/auto_reply/irrelevant/null |
| priority | ENUM | high/medium/low/null |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

#### emails（邮件记录）
| 字段 | 类型 | 说明 |
|------|------|------|

### 5.2 Agent 设计

| Agent | 模型 | 触发方式 | 工具依赖 |
|-------|------|---------|---------|
| Supervisor | - | 用户操作 / 定时 | LangGraph State |
| Scraper | - | Supervisor 分发 | Playwright |
| Sender | - | Supervisor 分发 | SMTP / SendGrid API |
| Monitor | - | 24/7 持续运行 | IMAP / Webhook |
| Classifier | GPT-4o-mini | 收到回复时 | - |
| Responder | GPT-4o | 月度追发/节假日 | SMTP |

### 5.3 关键技术决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 邮件发送 | SMTP 直发（非 SendGrid API） | 用户用自己的邮箱，SMTP 最通用 |
| 邮件监控 | IMAP 轮询 | 用户邮箱，无法用 SendGrid Webhook |
| 浏览器自动化 | Playwright | 稳定性好，反检测能力强，5 平台兼容 |
| 数据库 | SQLite → PostgreSQL | MVP 用 SQLite 快速启动，生产切 PG |
| 任务队列 | Redis + APScheduler | 月度追发 + 节假日定时任务 |
| 密码存储 | Fernet 对称加密 | 邮箱密码需要可逆解密用于 SMTP 登录 |

## 9. 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 平台反爬 | 无法抓取邮箱 | Playwright 指纹伪装 + 代理 IP 池 + 限速 |
| 邮件进垃圾箱 | 回复率极低 | SPF/DKIM/DMARC 配置指导 + 预热发送 |
| 单邮箱被封 | 发送中断 | 多邮箱轮换 + 日限额控制 + 退信率监控 |
| IMAP 连接断开 | 回复检测延迟 | 断线自动重连 + 心跳检测 + 告警 |
| LLM 分类不准 | 误判意图 | 人工可修正分类 + 反馈闭环优化 Prompt |
